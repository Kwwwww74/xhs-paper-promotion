# LLM and Agent Positioning

Use this guide for papers about large language models, agents, RAG, multimodal systems, evaluation, alignment, tool use, planning, memory, code agents, model releases, or AI applications.

## Extract AI-Specific Paper Card

Capture these fields when supported by the input:

- `任务`: reasoning, retrieval, coding, planning, tool use, memory, evaluation, safety, multimodal understanding, data generation, or deployment
- `对象`: model, agent framework, dataset, benchmark, training method, inference method, application system, or analysis
- `技术路线`: prompting, fine-tuning, RL, synthetic data, retrieval, planning loop, tool calling, memory module, verifier, evaluator, multi-agent collaboration, or environment feedback
- `实验设置`: benchmarks, datasets, baselines, metrics, models, ablations, human evaluation, or deployment scenario
- `关键结果`: only concrete results stated in the paper or provided text
- `工程价值`: what builders can learn, reuse, compare, or avoid
- `研究价值`: what future papers can build on
- `限制`: benchmark scope, data leakage risk, model dependency, cost, latency, reproducibility, generalization, safety, or real-world reliability

## Common Angles

Choose angles that match the paper:

- `Agent workflow`: "它把 Agent 的执行流程拆得更清楚"
- `Tool use`: "重点不在模型更大，而在怎么调用工具"
- `RAG`: "它重新处理了检索、生成和评估之间的关系"
- `Evaluation`: "它指出现有评测可能没有测到真正能力"
- `Benchmark`: "它给某类能力提供了更可比较的测试场"
- `Memory`: "它讨论 Agent 如何保留和使用长期信息"
- `Planning`: "它关注复杂任务中分解、执行和纠错"
- `Code agent`: "它适合关注自动编程和软件工程 Agent 的人看"
- `Multimodal`: "它把文本、图像、视频或交互信息放进同一个任务里看"
- `Alignment/safety`: "它提醒我们能力提升之外还要看可靠性和边界"
- `Model release`: "它的价值不只在参数规模，而在训练、评测或使用方式"

## Audience Positioning

Map the post to one primary audience:

- `AI 研究生`: emphasize literature value, method, experiment design, limitations
- `算法工程师`: emphasize reproducibility, benchmark setup, ablations, implementation hints
- `AI 产品经理`: emphasize scenario, workflow, user value, adoption constraints
- `创业者/独立开发者`: emphasize what can be borrowed for prototypes without overselling
- `泛 AI 关注者`: emphasize one clear concept and why it matters

## Title Patterns for AI Papers

Use specific nouns. Avoid generic "AI 又炸了".

- "Agent 不只是会调用工具"
- "这篇论文把 RAG 的问题讲透了"
- "LLM 评测可能测错了重点"
- "多智能体协作，到底强在哪？"
- "代码 Agent 的瓶颈不只在模型"
- "这篇适合做 Agent 选题入口"
- "别只看榜单，先看评测怎么设"
- "一个让 AI 更会规划的方法"
- "工具调用强不强，关键看这里"
- "RAG 的难点不只是检索"

## Body Framing

For AI papers, a good post usually answers:

1. This paper studies which AI capability or system problem?
2. Why is the problem important now?
3. What is the core technical idea?
4. How did the authors evaluate it?
5. What should readers take away?
6. What should not be overread?

## Claim Boundaries

Use these replacements:

- "模型具备推理能力" -> "模型在这些推理任务上表现更好"
- "Agent 会自主完成任务" -> "Agent 在设定环境中完成了多步任务"
- "超过人类" -> "在某个 benchmark 指标上超过了给定基线"
- "解决幻觉" -> "降低了特定设置下的错误或不一致"
- "通向 AGI" -> "为通用智能体研究提供了一个可讨论方向"
- "可直接落地" -> "有工程参考价值，但还要看成本、稳定性和场景"
- "开源可复现" -> "如果代码、模型或数据已公开，再说明可复现条件"
